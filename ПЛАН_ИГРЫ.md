# План создания игры для Яндекс Игр

## Часть 1. Популярные темы и тренды (исследование)

### Что интересно игрокам в 2024–2025

**Жанры с ростом:**
- **Пазлы и блоки** — самый большой рост: match-3, блок-пазлы (типа «Блоки и точка»), головоломки. Игроки любят короткие сессии и понятные правила.
- **Merge-игры** — слияние предметов (merge-2, merge-3), часто в связке с фермой, таверной, замком. Примеры на Яндекс Играх: «Мой Замок: Слияние и История», «Мир домовят», Merge Dale, Merge Mercs.
- **Гибридный казуал** — простая механика + элементы RPG, крафта, выживания. Авто-стрельба, авто-апгрейд, минимум сложного управления.
- **Хайперказуал** — очень простые механики, быстрый вход в игру, много повторов (одна механика на игру).
- **Idle / кликеры** — прогресс «сам идёт», игрок периодически заходит и улучшает. Пример: Firestone на Яндекс Играх.

### Что хорошо заходит на Яндекс Играх (по рейтингу и отзывам)

| Тема / жанр           | Примеры на платформе              | Почему работает        |
|----------------------|-----------------------------------|-------------------------|
| Merge + история/мир  | Мой Замок, Мир домовят            | Уют, прогресс, слияние  |
| Атмосферные/уютные   | Lamplighter                       | Настроение, расслабление|
| Блок-пазлы           | Блоки и точка, Соедини Блоки Пазл | Простота, «ещё один уровень» |
| Стратегия/стройка    | Пиратские корабли                 | Строй и сражайся        |
| Классика (пасьянс)   | Клондайк                          | Знакомый формат         |
| Tower Defense + merge| Merge Mercs                       | Гибрид механик          |
| Аркады               | Tank Stars, Little Big Snake      | Быстрые сессии          |
| «Найди отличия»      | Отличия онлайн                    | Спокойный челлендж      |

### Вывод по темам

- **Стабильно востребованы:** блок-пазлы, merge, уютная/спокойная атмосфера, короткие сессии, понятная цель («собери линию», «слияй до максимума», «пройди уровень»).
- **Удобно для одного разработчика:** игры на HTML5 без тяжёлого движка — блок-пазл, merge-2, кликер, аркада (змейка, breakout), «найди отличия». Минимум анимаций и контента на старте, максимум геймплея.

## Часть 2. Выбор концепции игры

### Идея: «Блок-пазл с уютной темой» (условное название: «Уютные блоки»)

**Почему эта концепция:**
1. **Блок-пазлы** — в топе и по трендам, и по рейтингу на Яндекс Играх («Блоки и точка», «Соедини Блоки Пазл»).
2. **Уютная тема** (домики, природа, животные, интерьер) совпадает с трендом cozy и с успешными играми (Мир домовят, Lamplighter).
3. **Реализуемо одним человеком:** механика известна, можно сделать на Canvas + JS без движка, процедурные уровни дают «бесконечный» контент.
4. **Хорошо стыкуется с Яндекс Играми:** короткие сессии, реклама между уровнями, лидерборд по очкам/уровням, сохранения через SDK.

**Краткое описание геймплея:**
- Игрок получает фигуры из блоков (тетрисообразные) и ставит их на сетку.
- Цель: заполнять **полные линии** (или области) — они исчезают, дают очки.
- Уровень пройден, когда набрано нужное количество очков или очищено N линий.
- Можно добавить: комбо, бонусные фигуры, дневные задания, простой «уютный» сюжет (например, строительство деревни/домика).

**Целевая платформа:** браузер (десктоп + мобильные), затем публикация в каталоге Яндекс Игр с подключением SDK.

## Часть 3. План разработки (для себя)

### Этап 0. Подготовка (1–2 дня)
- [ ] Выбрать стек: **HTML5 + Canvas + ванильный JavaScript** (или лёгкий фреймворк типа Phaser, если решим усложнять).
- [ ] Настроить репозиторий и простую структуру проекта (папки: `src`, `assets`, `index.html`).
- [ ] Сделать минимальный цикл: отрисовка сетки и одна тестовая фигура, без геймплея.

### Этап 1. Ядро механики (3–5 дней)
- [ ] Сетка игрового поля (например, 10×10 или 8×8), отрисовка в Canvas.
- [ ] Набор фигур (тетрисо-подобные): генерация или фиксированный набор из 5–7 форм.
- [ ] Размещение фигуры по клику/тапу: выбор позиции, проверка «влезает ли».
- [ ] Удаление заполненных линий (и/или столбцов), начисление очков.
- [ ] Условие победы на уровне (N линий или N очков).
- [ ] Переход на следующий уровень (сложность можно пока линейно повышать: больше очков/линий).

### Этап 2. Игровой цикл и UX (2–3 дня)
- [ ] Экран меню: «Играть», «Продолжить» (если будет сохранение), «Настройки» (звук вкл/выкл).
- [ ] Экран уровня: счёт, цель уровня, кнопка «Перезапустить уровень».
- [ ] Экран победы уровня: очки, кнопка «Далее».
- [ ] Game Over: когда фигуры больше некуда ставить — показать итог, предложить повторить или выйти в меню.
- [ ] Адаптив: поле и фигуры масштабируются под экран (десктоп и мобильные).

### Этап 3. Полировка и контент (2–4 дня)
- [ ] Процедурная сложность: размер поля, типы фигур, цель по очкам/линиям в зависимости от номера уровня.
- [ ] Звуки: постановка фигуры, линия, победа, проигрыш (можно бесплатные паки).
- [ ] Простая графика: заменить «квадраты» на спрайты или стилизованные блоки (одна тема: домики, природа, животные).
- [ ] Экраны загрузки и подсказка при первом запуске (1–2 экрана: «ставь фигуры», «собирай линии»).

### Этап 4. Интеграция Яндекс Игр (2–3 дня)
- [ ] Подключить SDK: скрипт `sdk.js`, инициализация `YaGames.init()`.
- [ ] Сохранение прогресса: текущий уровень, счёт, настройки (звук) через API сохранений SDK.
- [ ] Реклама: показ рекламы между уровнями (по правилам платформы), не в принудительном порядке в середине уровня.
- [ ] Лидерборд: отправка очков (за сессию или лучший результат уровня) в таблицу лидеров SDK.
- [ ] Вызов `LoadingAPI.ready()` после загрузки ресурсов, как требует модерация.

### Этап 5. Публикация и тесты (1–2 недели)
- [ ] Проверка по чек-листу требований Яндекс Игр (интерфейс, реклама, возраст, стабильность).
- [ ] Регистрация в Консоли, создание черновика, загрузка билда (архив с игрой).
- [ ] Заполнение описания, скриншоты, промо-материалы, локализация (минимум RU, при желании EN).
- [ ] Включение монетизации (реклама) в Консоли, отправка на модерацию.
- [ ] После одобрения: следить за отзывами и метриками, планировать мелкие обновления (баги, баланс, новые уровни/фигуры).
