# Выгрузка «Уютные блоки» в Яндекс Игры

Краткий чек-лист и состав архива для публикации в [консоли Яндекс Игр](https://yandex.ru/dev/games/console/).

---

## Что уже сделано в проекте

- **SDK:** в `index.html` подключён скрипт `https://yandex.ru/games/sdk/v2` (для локальной проверки подойдёт; при выгрузке на платформу можно заменить на `/sdk.js`).
- **Инициализация:** `YandexSDK.init()` вызывается при старте в `main.js`.
- **LoadingAPI.ready():** вызывается после показа главного меню (`YandexSDK.notifyReady()` в `main.js`), как требуется для модерации.
- **Реклама:** rewarded (за доп. отмены) и interstitial (после победы уровня) через `YandexSDK.showRewarded` / `YandexSDK.showInterstitial`.

---

## Что сделать перед выгрузкой

### 1. Проверить игру локально

- Откройте игру в браузере (например, `python3 -m http.server 8000` и `http://localhost:8000`).
- Убедитесь, что меню открывается, можно начать игру, ставить фигуры, проходить уровень, заходить в магазин и достижения.
- Проверьте, что нет ошибок в консоли (F12 → Console).

### 2. (По желанию) Заменить URL SDK для платформы

При загрузке ZIP в консоль Яндекс Игр платформа подставляет свой SDK. Можно оставить в `index.html` текущий URL:

```html
<script src="https://yandex.ru/games/sdk/v2"></script>
```

Либо заменить на относительный путь (как в [документации](https://yandex.com/dev/games/doc/ru/requirements)):

```html
<script src="/sdk.js"></script>
```

Тогда в iframe на домене Яндекс Игр будет использоваться их версия SDK.

### 3. Собрать ZIP для выгрузки

**Включить в архив:**

- `index.html` (в корне архива)
- `styles.css`
- Папка `src/` со всеми `.js` файлами:
  - `config.js`, `shapes.js`, `game.js`, `progress.js`, `achievements.js`, `yandex-sdk.js`, `dialogue.js`, `render.js`, `particles.js`, `juice.js`, `dragdrop.js`, `main.js`
- Папка `assets/` целиком:
  - `assets/blocks/` — все `skin_*.png`, `pieces.png`
  - `assets/characters/` — все подпапки и изображения (1/, 2/, 3/, и т.д.)

**Не включать:**

- `node_modules/` (если появится)
- `.git/`
- `.cursor/`, `.claude/`
- Файлы документации: `CLAUDE.md`, `ПЛАН_ИГРЫ.md`, `YANDEX_UPLOAD.md` (в архив для платформы не обязательны)
- Служебные файлы и скрытые папки

**Структура ZIP:** в корне архива должны быть сразу `index.html`, `styles.css`, папки `src/` и `assets/`. Не кладите всё в одну подпапку (например, не должно быть `test-game/index.html` в архиве — иначе игра не откроется).

**Пример сборки в терминале (macOS/Linux):**

```bash
cd /Users/lleeeuuuuanchik/projects/test-game
zip -r uyutnye-bloki.zip index.html styles.css src/ assets/ -x "*.DS_Store" -x ".git/*" -x "*.md" -x ".cursor/*" -x ".claude/*"
```

Проверьте: распакуйте `uyutnye-bloki.zip` в пустую папку и откройте `index.html` из неё — игра должна запускаться.

## Публикация в консоли

1. Зайдите в [Консоль Яндекс Игр](https://yandex.ru/dev/games/console/).
2. Создайте новую игру или откройте черновик.
3. Загрузите собранный ZIP (файл игры).
4. Заполните обязательные поля: название, описание, категория, картинки (иконка, превью по требованиям консоли).
5. Сохраните черновик и отправьте игру на модерацию.

После прохождения модерации игра появится в каталоге Яндекс Игр.

---

## Требования платформы (напоминание)

- Вызов **LoadingAPI.ready()** после загрузки — реализован в `YandexSDK.notifyReady()` после показа меню.
- Работа в iframe на домене Яндекс Игр — игра не использует `localStorage` в критичных для запуска местах; прогресс сохраняется в `localStorage` (на стороне пользователя).
- Реклама — подключена rewarded и fullscreen; при отсутствии SDK реклама не показывается, игра продолжает работать.

Если появятся замечания модерации (например, по контенту или скриншотам), их нужно будет учесть и загрузить обновлённый ZIP.
